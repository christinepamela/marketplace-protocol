## 🚨 CRITICAL RULES & PRINCIPLES

### Rule 1: Zero Assumptions
**ALWAYS ask for files before making changes**
- Request to see existing files
- Check current implementation
- Understand context before proposing changes
- Never assume structure or naming

### Rule 2: Backward Compatibility
**Every change must maintain existing functionality**
- Test after each change
- Don't break existing features
- Keep old APIs working
- Minimal, surgical changes only

### Rule 3: Eliminate Tech Debt
**When possible, fix issues at the root**
- Don't add placeholders
- No mock/fake data in production paths
- Fix problems properly, not with workarounds
- Document decisions in code comments

### Rule 4: Test Before Push
**Always test changes before committing**
- Run layer tests: `npm run test:layer0` through `test:layer6`
- Manual testing for UI changes
- Check backward compatibility
- Verify nothing broke

### Rule 5: Token Management
**Stop at 75% token usage**
- Update handover document continuously
- Create new artifact at 75%
- Clear handoff with no information loss
- Hard stop at 80% (safety margin)
