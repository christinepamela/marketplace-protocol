## 📋 Project Overview

**Rangkai Protocol** is a decentralized marketplace protocol designed to globalize small businesses by connecting manufacturers/artisans with buyers through a non-custodial, Bitcoin-native infrastructure.

### Core Principles
- **Non-Custodial**: No platform holds funds; 7-day escrow with auto-release
- **Bitcoin-Native**: Lightning Network & on-chain settlement
- **Decentralized**: Federated search across multiple marketplaces
- **Privacy-First**: DID-based identity, optional KYC
- **Fair Fees**: 3% protocol fee, 0-5% client marketplace fee

## 🔑 Key Technical Decisions

### Authentication
- **JWT tokens** for user sessions (24h expiry)
- **API keys** for server-to-server (hashed with HMAC-SHA256)
- **DID-based identity** (did:rangkai:uuid format)

### Error Handling
- **Standardized error codes** (FORBIDDEN, ORDER_NOT_FOUND, etc.)
- **HTTP status codes** via ERROR_STATUS_MAP
- **Consistent JSON responses** with error details

### Escrow
- **7-day hold period** (configurable per order)
- **Auto-release** after delivery + 7 days (no dispute)
- **Non-custodial** - protocol tracks, doesn't hold funds

### Rate Limiting
- **100 requests/hour** per client ID or IP
- **IPv6 compatible** (express-rate-limit built-in)
- **Bypass in test environment**

## ⚠️ Important Notes for Future Development

### Backward Compatibility
- **Always ask for existing files** before creating new ones
- **Match existing service method signatures** exactly
- **Use existing error codes** before adding new ones
- **Follow established patterns** (e.g., `getUserDid()` helper)

### Database Schema
- **JSONB fields** for flexible data (Layer 0 identities, Layer 1 products)
- **UUID primary keys** everywhere
- **DID format**: `did:rangkai:${uuid}` (must pass regex check)
- **Timestamptz** for all dates

### Testing Pattern
- **Create test data** → Run tests → **Clean up** (in finally block)
- **Use actual Supabase** (no mocks)
- **Generate unique IDs** with uuidv4()
- **Check response.ok AND result.success**
