### Why Separate Logistics?

**Traditional E-commerce (Locked-In):**
```
Amazon Marketplace → Amazon Logistics (Monopoly)
Lazada → Lazada Express (Monopoly)
```

**Rangkai Protocol (Open Competition):**
```
ANY Marketplace → Logistics Pool → 100+ providers compete
```

**Benefits:**
- **Vendors:** Get competitive pricing, not forced into expensive regional monopolies
- **Buyers:** Can choose trusted providers, override vendor's choice
- **Providers:** Access to global clients, not locked to one platform
- **Small Businesses:** Fair shipping costs enable global trade

### Quote System Details

**Quote Lifecycle:**
```
PENDING → Provider submitted, awaiting decision
    ├─→ ACCEPTED → Vendor/buyer selected, create shipment
    ├─→ REJECTED → Vendor chose another provider
    └─→ EXPIRED → Past valid_until timestamp
```

**Quote Types:**
- **Product-level quotes:** Pre-negotiated, valid 30–90 days (vendor configurable)
- **Order-level quotes:** On-demand, valid 24–48 hours (provider configurable)
- **Expiry notifications:** Sent 7 days before expiry

### Core Workflows

**Provider Registration:**
1. Visit logistics-marketplace.com → click "Register as Provider"
2. Complete KYC verification
3. Set up profile (business name, service regions, shipping methods, insurance)
4. Submit → redirect to dashboard

**Provider Quotes Order:**
1. Dashboard → "Available Opportunities"
2. Filter by region/weight/method
3. Submit quote (method, price, days, insurance, validity)
4. Wait for vendor acceptance

**Buyer Override at Checkout:**
1. Buyer sees vendor's chosen provider at checkout
2. Clicks "Choose My Own Provider"
3. Browses logistics pool, selects preferred provider
4. Vendor notified, checkout completes with buyer's provider
