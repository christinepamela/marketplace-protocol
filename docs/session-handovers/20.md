### 🐛 Bug #1: Shipment Details 404 (FIXED)

**Issue:**
```
GET /logistics/shipments/:id → 404 "Resource not found"
- Shipment exists in database ✅
- Create shipment works ✅
- Shipment appears in list ✅
- Clicking shipment → 404 error ❌
```

**Root Cause:**
- **SDK called:** `GET /logistics/shipments/:id/tracking`
- **Backend expected:** `GET /logistics/shipments/:id/history`
- **Path mismatch:** `/tracking` vs `/history`

**Fix Applied:**
```typescript
// File: packages/sdk/src/modules/logistics.ts (line ~280)

// BEFORE:
async getTrackingHistory(shipmentId: string): Promise<TrackingEvent[]> {
  return this.http.get(`/logistics/shipments/${shipmentId}/tracking`);
}

// AFTER:
async getTrackingHistory(shipmentId: string): Promise<TrackingEvent[]> {
  return this.http.get(`/logistics/shipments/${shipmentId}/history`);
}
```

**Result:** ✅ Shipment details page now loads correctly

### 🐛 Bug #2: Update Tracking 404 (FIXED)

**Issue:**
```
POST /logistics/shipments/:id/tracking → 404 "Resource not found"
- Update form loads ✅
- Submit button clicked ✅
- Request fails with 404 ❌
```

**Root Cause:**
- **SDK called:** `POST /logistics/shipments/:id/tracking`
- **Backend expected:** `PUT /logistics/shipments/:id/status`
- **Two mismatches:** Wrong HTTP method (`POST` vs `PUT`) + Wrong endpoint (`/tracking` vs `/status`)

**Fix Applied:**
```typescript
// File: packages/sdk/src/modules/logistics.ts (line ~261)

// BEFORE:
async updateTracking(
  shipmentId: string,
  update: UpdateTrackingRequest
): Promise<{ message: string }> {
  return this.http.post(`/logistics/shipments/${shipmentId}/tracking`, update);
}

// AFTER:
async updateTracking(
  shipmentId: string,
  update: UpdateTrackingRequest
): Promise<{ message: string }> {
  return this.http.put(`/logistics/shipments/${shipmentId}/status`, update);
}
```

**Result:** ✅ Tracking updates now work perfectly
