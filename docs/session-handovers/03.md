## 🔑 Technical Reference

### Environment Variables

```bash
SUPABASE_URL=...
SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_KEY=...
JWT_SECRET=<64-char-hex>
API_KEY_SALT=<64-char-hex>
PROOF_SIGNING_KEY=<128-char-hex>
```

### Key Patterns

| Pattern | Implementation |
|---|---|
| Route structure | All routes match existing Layer services exactly |
| Authentication | JWT for users, API keys for servers |
| Validation | Zod schemas for request/response |
| Errors | Standardized `ApiError` with codes |
| Updates | Partial updates merge with existing data |
| Deletes | Soft deletes (status changes, not removal) |

### Testing

Each route file has a corresponding test file under `scripts/`. Tests create real data, verify responses, and check auth.

```bash
npm run test:identity-routes
npm run test:catalog-routes
```
