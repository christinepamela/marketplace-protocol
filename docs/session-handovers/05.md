## 1. Protocol Overview

**Rangkai Protocol** is a decentralized marketplace protocol designed to globalize small businesses by connecting manufacturers and artisans with buyers through a non-custodial, Bitcoin-native infrastructure.

### Core Principles

- **Non-Custodial**: No platform holds funds; 7-day escrow with auto-release
- **Bitcoin-Native**: Lightning Network & on-chain settlement
- **Decentralized**: Federated search across multiple marketplaces
- **Privacy-First**: DID-based identity, optional KYC
- **Fair Fees**: 3% protocol fee, 0–5% client marketplace fee

## 7. Debugging Journey & Solutions

> These issues repeat across phases — read carefully before starting Phase 7.

### Issue 1 — Wrong Validation Middleware Name

**Symptom**: `validate is not a function`  
**Root Cause**: Used `validate()` instead of `validateBody()`  
**Fix**: Change all instances to `validateBody()`  
**Lesson**: Always check existing middleware exports before using them.

---

### Issue 2 — Missing `getUserDid` Helper

**Symptom**: `Module has no exported member 'getUserDid'`  
**Root Cause**: Function didn't exist in `utils.ts`  
**Fix**: Added the helper manually:

```typescript
export function getUserDid(req: Request): string {
  if (!req.user?.did) {
    throw new UnauthorizedError('User DID not found in token');
  }
  return req.user.did;
}
```

**Lesson**: Check if helpers exist before importing them.

---

### Issue 3 — Wrong Permission Constants

**Symptom**: `Insufficient permissions`  
**Root Cause**: Used `product:write` instead of `catalog:write`  
**Fix**: Check `PERMISSIONS` object in `auth.ts` and use the correct names.

Correct permission names:
- `catalog:write` (not `product:write`)
- `orders:write` (not `order:write`)
- `trust:dispute`
- `trust:rate`

**Lesson**: Always reference existing permission constants — never guess.

---

### Issue 4 — Wrong Product Schema Structure

**Symptom**: `Request body validation failed` with many missing fields  
**Root Cause**: Assumed a simple product structure  
**Fix**: Copy the exact schema from `scripts/test-catalog-routes.ts`

```typescript
{
  vendorDid: string,
  clientId: string,
  category: {
    primary: 'footwear' | ...,
    subcategory: string,
    tags: string[]
  },
  basic: {
    name: string,              // NOT title
    images: { primary: string },
    condition: 'new' | 'used'
  },
  pricing: {
    moq: number,
    basePrice: { amount, currency }
  },
  logistics: {
    weight: { value, unit },
    dimensions: { length, width, height, unit },
    originCountry: string,
    leadTime: number
  }
}
```

**Lesson**: When in doubt, copy from working tests exactly.

---

### Issue 5 — Wrong Order Schema Structure

**Symptom**: Multiple missing required fields  
**Root Cause**: Assumed single-product order; API uses cart-style  
**Fix**: Copy the exact structure from `scripts/test-order-routes.ts`

```typescript
{
  vendorDid: string,
  clientId: string,
  type: 'wholesale' | 'sample' | 'custom',
  items: [{
    productId: string,
    productName: string,
    quantity: number,
    pricePerUnit: { amount, currency },
    totalPrice: { amount, currency }
  }],
  shippingAddress: {
    name: string,           // NOT recipientName
    addressLine1: string,   // NOT line1
    phone: string,
    city: string,
    postalCode: string,
    country: string
  },
  paymentMethod: 'stripe' | 'bitcoin_onchain' | ...
}
```

**Lesson**: Always check the existing API structure before assuming.

---

### Issue 6 — Routes Hanging (30s Timeout) ⚠️ Critical

**Symptom**: All dispute/rating requests timeout with no response  
**Root Cause**: Missing parentheses when calling `authenticate` middleware

```typescript
// ❌ WRONG
import { authenticate() } from '../middleware/auth.middleware';
router.post('/', authenticate, ...)

// ✅ CORRECT
import { authenticate } from '../middleware/auth.middleware';
router.post('/', authenticate(), ...)
```

Express received the function definition instead of calling it as middleware, causing an infinite wait.  
**Lesson**: Import the function, call it with `()` when using as middleware.

---

### Issue 7 — Stats Route Conflict

**Symptom**: `Invalid UUID: stats`  
**Root Cause**: `/disputes/stats` matched `/disputes/:id` first  
**Fix**: Move the stats route **before** the `:id` route

```typescript
router.get('/stats', ...)   // Specific routes first ✅
router.get('/:id', ...)     // Parameterized routes last ✅
```

**Lesson**: Express matches routes in order — specific paths always before parameters.

---

### Issue 8 — TypeScript Strict Mode

**Symptom**: `supabase might be undefined`  
**Root Cause**: `express.d.ts` had `supabase?: SupabaseClient` (optional)  
**Fix**: Changed to required — `supabase: SupabaseClient`  
**Lesson**: If middleware guarantees a property on every request, mark it required in types.

## 10. Testing Patterns

### Test File Structure

```typescript
// 1. Imports
import { v4 as uuidv4 } from 'uuid';

// 2. Constants
const API_BASE = 'http://localhost:3000/api/v1';

// 3. Test state
let testUserId: string;
let authToken: string;

// 4. Helper functions
async function apiRequest(method, path, body?, token?) { ... }

// 5. Setup
async function setupTestData() { ... }

// 6. Individual tests
async function testCreateResource() { ... }

// 7. Cleanup
async function cleanup() { ... }

// 8. Runner
async function runTests() {
  await setupTestData();
  await testCreateResource();
  await cleanup();
}

runTests().catch(console.error);
```

### Setup Pattern

```typescript
async function setupTestData() {
  const buyerRes = await apiRequest('POST', '/identity/register', {
    type: 'kyc',
    clientId: 'test-marketplace',
    publicProfile: {
      displayName: 'Test Buyer',
      country: 'US',
      businessType: 'buyer'
    }
  });
  buyerDid = buyerRes.data.did;

  const { TokenManager } = await import('../src/api/core/auth');
  buyerToken = TokenManager.generateToken({
    sub: buyerDid,
    did: buyerDid,
    type: 'user',
    permissions: ['orders:read', 'orders:write']
  });
}
```

### Test a Success Case

```typescript
async function testCreateOrder() {
  try {
    const response = await apiRequest('POST', '/orders', { /* full data */ }, buyerToken);
    if (response.success && response.data.orderId) {
      logTest('Create order', true, `Order ID: ${response.data.orderId}`);
      testOrderId = response.data.orderId;
    } else {
      logTest('Create order', false, 'Invalid response');
    }
  } catch (error) {
    logTest('Create order', false, error.message);
  }
}
```

### Test a Forbidden Case

```typescript
async function testVendorCannotFile() {
  try {
    await apiRequest('POST', '/disputes', { ... }, vendorToken);
    logTest('Vendor blocked', false, 'Should have been forbidden');
  } catch (error) {
    if (error.message.includes('buyer') || error.message.includes('FORBIDDEN')) {
      logTest('Vendor blocked', true);
    } else {
      logTest('Vendor blocked', false, error.message);
    }
  }
}
```

### Cleanup Pattern

```typescript
async function cleanup() {
  console.log('Cleaning up test data...');
  // Delete in reverse order of dependencies
  if (testOrderId) await supabase.from('orders').delete().eq('id', testOrderId);
  if (testProductId) await supabase.from('products').delete().eq('id', testProductId);
  console.log('✓ Cleanup complete');
}
```
