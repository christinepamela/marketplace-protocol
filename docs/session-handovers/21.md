## 🔍 FEDERATED SEARCH DEEP DIVE

**WHY IT'S THE KILLER FEATURE:**

Traditional marketplaces = Walled gardens
- Amazon products only on Amazon
- eBay products only on eBay
- Each marketplace = separate search index

Rangkai Protocol = Federated search
- **ONE search indexes ALL marketplaces**
- Buyer on Marketplace A can find products from Marketplace B, C, D
- Vendors list ONCE, visible EVERYWHERE

### **How It Works (Layer 1):**

```
┌─────────────────────────────────────────────────────────────┐
│ PRODUCT CREATION (Vendor)                                  │
└─────────────────────────────────────────────────────────────┘
   ├─ Vendor creates product on Marketplace A
   ├─ Product saved to `products` table (canonical)
   ├─ Trigger: `sync_product_to_index()`
   └─ Product automatically indexed

┌─────────────────────────────────────────────────────────────┐
│ INDEX SYNCHRONIZATION (Automatic)                          │
└─────────────────────────────────────────────────────────────┘
   ├─ Database trigger fires
   ├─ Product data copied to `product_index` table
   ├─ Full-text search fields populated:
   │  ├─ `search_vector` (PostgreSQL tsvector)
   │  ├─ `name_vector` (product name)
   │  └─ `description_vector` (description)
   ├─ Metadata extracted:
   │  ├─ Category
   │  ├─ Price
   │  ├─ Vendor DID
   │  └─ Client ID (marketplace identifier)
   └─ Index updated in real-time (<1 second)

┌─────────────────────────────────────────────────────────────┐
│ SEARCH QUERY (Buyer on ANY Marketplace)                    │
└─────────────────────────────────────────────────────────────┘
   ├─ Buyer types: "leather backpack"
   ├─ Query sent to Layer 1 search.service.ts
   ├─ PostgreSQL full-text search:
   │  ├─ SELECT * FROM product_index
   │  ├─ WHERE search_vector @@ to_tsquery('leather & backpack')
   │  └─ ORDER BY ts_rank(search_vector, query) DESC
   ├─ Results ranked by:
   │  ├─ Relevance score (text matching)
   │  ├─ Vendor reputation (joined from Layer 0)
   │  ├─ Price (if buyer filtered)
   │  └─ Location (shipping cost estimate)
   └─ Returns top 50 results across ALL marketplaces

┌─────────────────────────────────────────────────────────────┐
│ RESULTS DISPLAY (Cross-Marketplace)                        │
└─────────────────────────────────────────────────────────────┘
   ├─ Product A: From Marketplace A (Malaysia)
   │  ├─ Handmade Leather Backpack
   │  ├─ $150 USD + $12 shipping
   │  └─ Vendor: 4.8★ (234 reviews)
   ├─ Product B: From Marketplace B (Indonesia)
   │  ├─ Genuine Leather Backpack
   │  ├─ $140 USD + $15 shipping
   │  └─ Vendor: 4.6★ (89 reviews)
   └─ Product C: From Marketplace A (Malaysia)
      ├─ Premium Leather Backpack
      ├─ $180 USD + $12 shipping
      └─ Vendor: 4.9★ (567 reviews)

RESULT: Buyer gets best options from ALL marketplaces! ✨
```

```
- Generated uniquely per order (HD wallet)
- Protocol controls private keys
- Released only after delivery OR dispute resolution

Security:
- Multi-sig possible (protocol + trusted party)
- Time-locked transactions (advanced)
- Dispute resolution can reverse payment
```
