# PART 1: PROTOCOL OVERVIEW

## 1. What is Rangkai Protocol?

**Rangkai Protocol** is a decentralized marketplace protocol designed to globalize small businesses by connecting manufacturers and artisans with buyers through a non-custodial, Bitcoin-native infrastructure.

### Core Principles

✅ **Non-Custodial:** No platform holds funds; 7-day escrow with auto-release  
✅ **Bitcoin-Native:** Lightning Network & on-chain settlement  
✅ **Decentralized:** Federated search across multiple marketplaces  
✅ **Privacy-First:** DID-based identity, optional KYC  
✅ **Fair Fees:** 3% protocol fee, 0-5% client marketplace fee  

### Key Features

- **Identity & Reputation** (Layer 0) - KYC, Nostr, or Anonymous identities
- **Product Discovery** (Layer 1) - Federated search across marketplaces
- **Order Management** (Layer 2) - Non-custodial escrow with state machine
- **Logistics** (Layer 3) - Shipping quotes, tracking, provider network
- **Trust & Compliance** (Layer 4) - Disputes, ratings, sanctions screening
- **Governance** (Layer 6) - 3-of-3 multisig for protocol parameters

## 18. Dos and Don'ts

### ✅ DOs

**Code Organization:**
- ✅ Keep business logic in `core/` services
- ✅ Keep API routes in `api/routes/`
- ✅ Use consistent file naming
- ✅ Follow existing patterns

**API Development:**
- ✅ Always validate request bodies with Zod
- ✅ Use `authenticate()` for protected routes
- ✅ Call middleware functions: `authenticate()`
- ✅ Return consistent response format: `{ success, data }`
- ✅ Use proper HTTP status codes

**Error Handling:**
- ✅ Use `ApiError` classes
- ✅ Pass errors to `next(error)`
- ✅ Provide helpful error messages
- ✅ Include error details for debugging

**Testing:**
- ✅ Test after each feature
- ✅ Clean up test data
- ✅ Test error cases
- ✅ Run full test suite before committing

**WebSocket:**
- ✅ Emit events AFTER database operations
- ✅ Fetch updated entity before emitting
- ✅ Use fire-and-forget (don't block on WebSocket)
- ✅ Handle WebSocket errors gracefully

**Git:**
- ✅ Commit frequently
- ✅ Write descriptive commit messages
- ✅ Test before committing
- ✅ Push working code only

### ❌ DON'Ts

**Code Anti-Patterns:**
- ❌ Don't hardcode values (use config)
- ❌ Don't import with `()`: `import { func() }` ❌
- ❌ Don't use `validate`, use `validateBody`
- ❌ Don't access `req.user.did` directly, use `getUserDid(req)`

**Routing:**
- ❌ Don't put parameterized routes before specific routes
  ```typescript
  // ❌ WRONG ORDER
  router.get('/:id', ...)      // Matches '/stats'!
  router.get('/stats', ...)

  // ✅ CORRECT ORDER
  router.get('/stats', ...)
  router.get('/:id', ...)
  ```

**Error Handling:**
- ❌ Don't catch errors and send response directly
- ❌ Don't swallow errors silently
- ❌ Don't expose internal error details to clients

**Security:**
- ❌ Don't commit `.env` file
- ❌ Don't use weak JWT secrets
- ❌ Don't disable authentication in production
- ❌ Don't trust user input without validation

**Testing:**
- ❌ Don't skip tests
- ❌ Don't commit broken tests
- ❌ Don't test against production database
- ❌ Don't leave test data in database

**WebSocket:**
- ❌ Don't emit events before database save
- ❌ Don't use stale data in events
- ❌ Don't block API on WebSocket errors
- ❌ Don't forget null checks: `updatedOrder!.id`

## 19. Common Patterns

### API Route Pattern
```typescript
router.post(
  '/endpoint',
  authenticate(),                    // Auth middleware
  validateBody(schema),              // Validation
  async (req, res, next) => {
    try {
      const userDid = getUserDid(req);  // Get user
      const service = new Service(req.supabase);

      // Business logic
      const result = await service.method(data);

      // Emit event (if applicable)
      emitEvent('event.name', result);

      // Response
      res.status(200).json({
        success: true,
        data: result
      });
    } catch (error) {
      next(error);  // Pass to error handler
    }
  }
);
```

### Service Pattern
```typescript
export class SomeService {
  constructor(private supabase: SupabaseClient) {}

  async doSomething(params): Promise<Result> {
    // Validate
    if (!params.id) {
      throw new ValidationError('ID required');
    }

    // Database operation
    const { data, error } = await this.supabase
      .from('table')
      .insert(params)
      .select()
      .single();

    if (error) {
      throw new InternalError('Database error', { error });
    }

    return data;
  }
}
```

### Event Emission Pattern
```typescript
// After successful operation
await service.updateOrder(id, updates);

// Fetch updated entity
const updatedOrder = await service.getOrderById(id);

// Emit event with fresh data
emitOrderEvent('order.updated', updatedOrder);
```

### Error Handling Pattern
```typescript
try {
  // Operation
  const result = await service.method();
} catch (error) {
  // Let error middleware handle it
  next(error);
}

// In error middleware
if (error instanceof ApiError) {
  res.status(error.statusCode).json(error.toJSON());
} else {
  res.status(500).json({ error: 'Internal error' });
}
```

## 20. Debugging Tips

### Common Issues & Solutions

**Issue 1: "Cannot find module"**
```typescript
// ❌ Wrong import
import { func() } from './module';

// ✅ Correct import
import { func } from './module';
// Then call: func()
```

**Issue 2: "Property 'did' does not exist"**
```typescript
// ❌ Direct access
const userDid = req.user.did;

// ✅ Use helper
const userDid = getUserDid(req);
```

**Issue 3: "authenticate is not a function"**
```typescript
// ❌ Imported with ()
import { authenticate() } from '../middleware/auth.middleware';

// ✅ Correct import
import { authenticate } from '../middleware/auth.middleware';
router.post('/', authenticate(), ...)
```

**Issue 4: WebSocket events not received**
```typescript
// ❌ Emitting before save
emitEvent('created', entity);
await service.create(entity);

// ✅ Emit after save
await service.create(entity);
const saved = await service.getById(id);
emitEvent('created', saved);
```

**Issue 5: TypeScript null errors**
```typescript
// ❌ Possible null
const name = order.buyer.name;

// ✅ Add null check or assertion
const name = order?.buyer?.name;
// or
const name = order!.buyer!.name;  // If you know it exists
```

### Debugging Commands
