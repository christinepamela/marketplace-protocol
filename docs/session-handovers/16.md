## 🐛 ISSUES RESOLVED

### Issue #18: SDK Not Sending Auth Token (CRITICAL) ✅ FIXED
**Problem:** SDK configuration used wrong property name  
**Root Cause:** Used `baseURL` and `getToken` when SDK expects `apiUrl` and `token`  
**Fix Applied:**
```typescript
// lib/sdk.ts
export const sdk = new RangkaiSDK({
  apiUrl: 'http://localhost:3000',  // Correct property
  sandbox: true,
  token: typeof window !== 'undefined' 
    ? localStorage.getItem('rangkai_token') || undefined 
    : undefined  // Correct approach
})
```
**Status:** ✅ RESOLVED - Dashboard now loads data successfully

### Issue #20: Opportunities Endpoint Returns Incomplete Data ✅ FIXED
**Problem:** Backend returned raw order data without weight, dimensions, or product details

## 🔑 KEY LEARNINGS FROM SESSION 16

### 1. SDK Configuration
**Lesson:** Always verify SDK interface matches implementation  
**Issue:** Used `baseURL` when SDK expects `apiUrl`  
**Fix:** Check SDK source code or examples before configuring

### 2. Token Management
**Lesson:** JWT tokens in client apps need proper passing  
**Solution:** Use `token` property with localStorage getter  
**Future:** Consider refresh token flow for production

### 3. API Data Completeness
**Lesson:** Backend endpoints may return incomplete data  
**Discovery:** Opportunities endpoint had no tests, no spec for response format  
**Fix:** Enhanced to join related tables and calculate derived fields  
**Prevention:** Always add tests for new endpoints

### 4. Frontend Error Handling
**Lesson:** Use optional chaining for API data  
**Pattern:** `opportunity.dimensions?.length_cm || 0`  
**Why:** API data structure may not match TypeScript types exactly

### 5. Testing Strategy
**Lesson:** Manual testing reveals issues automated tests miss  
**Success:** Caught SDK config issue, incomplete API responses, UI bugs  
**Approach:** Test complete user flows, not just individual features
